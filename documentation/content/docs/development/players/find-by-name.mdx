## Find a player by name

Once you've initialized the `PlayerProvider`, you can look up a specific player by their display name:

### Synchronous

<Tabs items={['Kotlin', 'Java']}>
  <Tab value="Kotlin">
    <DynamicCodeBlock lang="kotlin" code='val player = playerProvider.findByName("Notch")' />
  </Tab>
  <Tab value="Java">
    <DynamicCodeBlock lang="java" code='PolocloudPlayer player = playerProvider.findByName("Notch");' />
  </Tab>
</Tabs>

<Callout type="warning">
  This method returns <code>null</code> if no player with the given name is found or if the player is offline.
</Callout>

### Asynchronous

If you prefer non-blocking access, use the asynchronous version:

<Tabs items={['Kotlin', 'Java']}>
  <Tab value="Kotlin">
    <DynamicCodeBlock
      lang="kotlin"
      code={`playerProvider.findByNameAsync("Notch").thenAccept { player ->
    if (player == null) return@thenAccept
    println("Found player: \${player.name} on \${player.currentServiceName}")
}`}
    />
  </Tab>
  <Tab value="Java">
    <DynamicCodeBlock
      lang="java"
      code={`playerProvider.findByNameAsync("Notch").thenAccept(player -> {
    if (player == null) return;
    System.out.println("Found player: " + player.name + " on " + player.currentServiceName);
});`}
    />
  </Tab>
</Tabs>

<Callout type="info">
  <code>findByNameAsync</code> returns a <code>CompletableFuture&lt;PolocloudPlayer&gt;</code>.  
  You can chain it with <code>thenAccept</code>, <code>thenApply</code>, or use it with Kotlin coroutines via <code>await()</code>.
</Callout>

## Return value

- **Sync:** <code>PolocloudPlayer</code> or <code>null</code>  
- **Async:** <code>CompletableFuture&lt;PolocloudPlayer&gt;</code> (which completes with a `PolocloudPlayer` or `null`)

The returned `PolocloudPlayer` object represents the **current state** of the requested player.  
If no player with the specified name exists or is online, the result is `null`.
