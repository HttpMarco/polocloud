## Find a service by name

Once you've initialized the `ServiceProvider`, you can look up a specific service by its name:

### Synchronous

<Tabs items={['Kotlin', 'Java']}>
  <Tab value="Kotlin">
    <DynamicCodeBlock lang="kotlin" code='val service = serviceProvider.find("lobby-1")' />
  </Tab>
  <Tab value="Java">
    <DynamicCodeBlock lang="java" code='Service service = serviceProvider.find("lobby-1");' />
  </Tab>
</Tabs>

<Callout type="warning">
  This method returns <code>null</code> if no service with the given name is found.
</Callout>

### Asynchronous

If you prefer non-blocking access, use the asynchronous version:

<Tabs items={['Kotlin', 'Java']}>
  <Tab value="Kotlin">
    <DynamicCodeBlock
      lang="kotlin"
      code={`serviceProvider.findAsync("lobby-1").thenAccept { service ->
    if (service == null) return@thenAccept
    println("Found service: \${service.name()}")
}`}
    />
  </Tab>
  <Tab value="Java">
    <DynamicCodeBlock
      lang="java"
      code={`serviceProvider.findAsync("lobby-1").thenAccept(service -> {
    if (service == null) return;
    System.out.println("Found service: " + service.name());
});`}
    />
  </Tab>
</Tabs>

<Callout type="info">
  <code>findAsync</code> returns a <code>CompletableFuture&lt;Service&gt;</code>.  
  You can chain it with <code>thenAccept</code>, <code>thenApply</code>, or use it with Kotlin coroutines via <code>await()</code>.
</Callout>

## Return value

- **Sync:** <code>Service</code> or <code>null</code>  
- **Async:** <code>CompletableFuture&lt;Service&gt;</code> (which completes with a `Service` or `null`)

The returned `Service` object represents the **current runtime state** of the requested service.  
If no service with the specified name exists or is active, the result is `null`.
