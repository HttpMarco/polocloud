## Find services by type

Use this method to get all services that belong to a specific `GroupType`.  
This is useful to filter services by their classification or role.

<Callout type="tip">
  For more details about the possible <code>GroupType</code> values, visit the <a href="../groups/group-types">Group Types</a> reference.
</Callout>

### Synchronous

<Tabs items={['Kotlin', 'Java']}>
  <Tab value="Kotlin">
    <DynamicCodeBlock lang="kotlin" code={`val services = serviceProvider.findByType(GroupType.SERVER)

services.forEach { service ->
    println("Service: \${service.name()} - Type: \${service.type}")
}`} />
  </Tab>
  <Tab value="Java">
    <DynamicCodeBlock lang="java" code={`List<Service> services = serviceProvider.findByType(GroupType.SERVER);

for (Service service : services) {
    System.out.println("Service: " + service.name() + " - Type: " + service.getType());
}`} />
  </Tab>
</Tabs>

<Callout type="info">
  The returned list may be empty if no services of the specified type are currently active.
</Callout>

### Asynchronous

For non-blocking retrieval, use the async version:

<Tabs items={['Kotlin', 'Java']}>
  <Tab value="Kotlin">
    <DynamicCodeBlock lang="kotlin" code={`serviceProvider.findByTypeAsync(GroupType.SERVER).thenAccept { services ->
    if (services.isEmpty()) return@thenAccept

    services.forEach { service ->
        println("Service: \${service.name()} - Type: \${service.type}")
    }
}`} />
  </Tab>
  <Tab value="Java">
    <DynamicCodeBlock lang="java" code={`serviceProvider.findByTypeAsync(GroupType.SERVER).thenAccept(services -> {
    if (services.isEmpty()) return;

    for (Service service : services) {
        System.out.println("Service: " + service.name() + " - Type: " + service.getType());
    }
});`} />
  </Tab>
</Tabs>

## Return value

- **Sync:** <code>List&lt;Service&gt;</code>  
- **Async:** <code>CompletableFuture&lt;List&lt;Service&gt;&gt;</code>

The returned list represents all services of the specified `GroupType` that are currently active.  
Services that are offline, unregistered, or provisioning are excluded.
