## Find a template by name

Once you've initialized the `TemplateProvider`, you can look up a specific template by its name:

### Synchronous

<Tabs items={['Kotlin', 'Java']}>
  <Tab value="Kotlin">
    <DynamicCodeBlock lang="kotlin" code='val template = templateProvider.find("lobby-template")' />
  </Tab>
  <Tab value="Java">
    <DynamicCodeBlock lang="java" code='Template template = templateProvider.find("lobby-template");' />
  </Tab>
</Tabs>

<Callout type="warning">
  This method returns <code>null</code> if no template with the given name is found.
</Callout>

### Asynchronous

If you prefer non-blocking access, use the asynchronous version:

<Tabs items={['Kotlin', 'Java']}>
  <Tab value="Kotlin">
    <DynamicCodeBlock
      lang="kotlin"
      code={`templateProvider.findAsync("lobby-template").thenAccept { template ->
    if (template == null) return@thenAccept
    println("Found template: \${template.name} (\${template.size()})")
}`}
    />
  </Tab>
  <Tab value="Java">
    <DynamicCodeBlock
      lang="java"
      code={`templateProvider.findAsync("lobby-template").thenAccept(template -> {
    if (template == null) return;
    System.out.println("Found template: " + template.name + " (" + template.size() + ")");
});`}
    />
  </Tab>
</Tabs>

<Callout type="info">
  <code>findAsync</code> returns a <code>CompletableFuture&lt;Template&gt;</code>.  
  You can chain it with <code>thenAccept</code>, <code>thenApply</code>, or use it with Kotlin coroutines via <code>await()</code>.
</Callout>

## Return value

- **Sync:** <code>Template</code> or <code>null</code>  
- **Async:** <code>CompletableFuture&lt;Template&gt;</code> (which completes with a `Template` or `null`)

The returned `Template` object represents the **current state** of the requested template.  
If no template with the specified name exists or is available, the result is `null`.
