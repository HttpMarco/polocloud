## What are Services?

In PoloCloud, a **Service** represents a running instance of your application or game server. Services are the actual executable units that handle player connections and game logic.

### Service Lifecycle

- **Booting** - Service is starting up
- **Running** - Service is active and accepting connections
- **Stopping** - Service is shutting down
- **Stopped** - Service is offline

### Finding Services

Discover services in your cloud infrastructure:

<DynamicCodeBlock lang="rust" code={`// Find all services
let mut service_provider = sdk.service_provider().lock().await?;
let all_services = service_provider.find_all_async().await?;

// Find services by name
let service_name = "lobby-1".to_string();
let services = service_provider.find_by_name_async(service_name).await?;

// Find services by group
let group_name = "lobby".to_string();
let group_services = service_provider.find_group_name_async(group_name).await?;

// Print results
println!("Found {} services:", all_services.len());
for service in all_services {
    println!("- {} ({}:{}) - State: {:?}", 
             service.group_name, 
             service.hostname, 
             service.port,
             service.state);
}`} />

### Service Properties

Services in the Rust SDK have the following properties:

| Property | Type | Description |
|----------|------|-------------|
| `group_name` | `String` | Parent group name |
| `id` | `u32` | Unique service ID |
| `hostname` | `String` | Service hostname |
| `port` | `u32` | Service port |
| `state` | `Option<ServiceState>` | Current state (Running, Stopped, etc.) |
| `server_type` | `GroupType` | Type of service |
| `properties` | `HashMap<String, String>` | Service properties |

<Callout type="tip">
**Best Practice**

Monitor service states regularly to ensure optimal performance and availability. Use the service properties to track resource usage and make scaling decisions.
</Callout>
