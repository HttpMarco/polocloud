## What are Groups?

In PoloCloud, a **Group** represents a logical collection or category of services. Groups help you organize and manage services by their function, environment, or any custom classification.

### Benefits of Using Groups

- **Logical Organization** - Categorize services by function or environment
- **Bulk Operations** - Manage multiple services simultaneously
- **Easy Discovery** - Filter and find services quickly
- **Scalability** - Set automatic scaling rules per group
- **Configuration** - Apply group-specific settings and templates

### Finding Groups

Discover existing groups in your cloud infrastructure:

<DynamicCodeBlock lang="rust" code={`// Find all groups
let mut group_provider = sdk.group_provider().lock().await?;
let all_groups = group_provider.find_all_async().await?;

// Find a specific group by name
let group_name = "lobby".to_string();
let lobby_group = group_provider.find_by_name_async(group_name).await?;

// Print results
println!("Found {} groups:", all_groups.len());
for group in all_groups {
    println!("- {} (min: {}MB, max: {}MB)", 
             group.name(), 
             group.minimum_memory(), 
             group.maximum_memory());
}`} />

### Group Properties

Groups in the Rust SDK have the following properties:

| Property | Type | Description |
|----------|------|-------------|
| `name()` | `&str` | Unique group name |
| `minimum_memory()` | `i32` | Minimum memory requirement in MB |
| `maximum_memory()` | `i32` | Maximum memory limit in MB |
| `minimum_online()` | `i32` | Minimum online services |
| `maximum_online()` | `i32` | Maximum online services |
| `properties()` | `&HashMap<String, String>` | Custom properties |
