## Install REST Module

Follow these steps to install and configure the REST Module for your PoloCloud instance.

<div className="text-center mb-12">
    <div className="w-20 h-1 bg-gradient-to-r from-transparent via-blue-500/50 to-transparent rounded-full mx-auto mb-6" />
    <div className="text-xl text-muted-foreground max-w-3xl mx-auto">
        Set up the REST API module for programmatic PoloCloud management
    </div>
</div>

### Installation Steps

1. **Download from GitHub Releases**

<DynamicCodeBlock
  lang="bash"
  code={`# Download the REST module JAR file from GitHub releases
wget https://github.com/polocloud/polocloud/releases/download/v3.0.0/rest-module-3.0.0.jar`}
/>

2. **Place in Local Modules Directory**

<DynamicCodeBlock
  lang="bash"
  code={`# Copy to your PoloCloud local modules directory
cp rest-module-3.0.0.jar /polocloud/local/modules/`}
/>

3. **Stop PoloCloud**

<DynamicCodeBlock
  lang="bash"
  code={`# Stop PoloCloud to load the module
systemctl stop polocloud`}
/>

## Configuration

### Basic Configuration

Configure the REST module through your PoloCloud configuration file:

<DynamicCodeBlock
  lang="json"
  code={`{
  "hostname": "0.0.0.0",
  "port": 8080,
  "securePort": 443,
  "sslEnabled": false,
  "certPath": "",
  "keyPath": ""
}`}
/>

### Advanced Configuration

<DynamicCodeBlock
  lang="json"
  code={`{
  "hostname": "0.0.0.0",
  "port": 8080,
  "securePort": 443,
  "sslEnabled": true,
  "certPath": "/path/to/certificate.crt",
  "keyPath": "/path/to/private.key"
}`}
/>

<Callout type="info">
**Local Hosting**

For local development, you can change the port to avoid conflicts:
- Set `"port": 3000` for local development
- Enable SSL with `"sslEnabled": true` and provide certificate paths
- Use `"hostname": "localhost"` to restrict access to local machine only
</Callout>

## Verification

### Test the Installation

<DynamicCodeBlock
  lang="bash"
  code={`# Check if the module is loaded
curl http://localhost:8080/api/health

# Expected response: {"status": "healthy"}`}
/>


<Callout type="info">
**Success Indicators**

- Module appears in PoloCloud logs
- Health endpoint responds with 200 status
- API endpoints are accessible
- No error messages in logs
</Callout>
