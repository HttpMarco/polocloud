## Overview

The **Notify Addon** provides real-time service state notifications to players with appropriate permissions. It automatically announces when services start, come online, or shut down, keeping your community informed about server status changes.

## Supported Platforms

The Notify Addon supports multiple proxy platforms:

- **Velocity** - High-performance proxy with MiniMessage support
- **BungeeCord** - Popular proxy solution with legacy text formatting

## Notification Types

The addon automatically sends notifications for the following service state changes:

### Service Starting
- **Trigger**: When a service enters the `STARTING` state
- **Permission**: `polocloud.addons.notify.receive.starting`
- **Default Message**: `§7Service §b%service% §7is starting! §8[§6⌛§8]`

### Service Online
- **Trigger**: When a service becomes `ONLINE`
- **Permission**: `polocloud.addons.notify.receive.online`
- **Default Message**: `§7Service §b%service% §7is now online! §8[§a✔§8]`

### Service Shutdown
- **Trigger**: When a service enters the `STOPPED` state
- **Permission**: `polocloud.addons.notify.receive.shutdown`
- **Default Message**: `§7Service §b%service% §7has been shut down. §8[§c✘§8]`

## Configuration Options

The Notify Addon creates a `notify-config.json` file with the following options:

### Basic Settings
- **`prefix`** - Message prefix for all notifications
- **`messages`** - Customizable notification messages

### Message Customization
All notification messages support placeholders and color codes:

- **`service_starting`** - Message when service starts
- **`service_online`** - Message when service comes online
- **`service_shutdown`** - Message when service shuts down
- **`no_permission`** - Permission denied message
- **`only_players`** - Console-only command message

## Installation

1. **Download** the Notify Addon from the PoloCloud releases
2. **Place** the addon JAR in your `/polocloud/addons/` directory
3. **Restart** PoloCloud to load the addon
4. **Configure** notification settings in the generated config file

## Configuration Example

Here's a complete example of the `notify-config.json` file:

```json
{
  "prefix": "§8[§bPoloCloud§8] ",
  "messages": {
    "no_permission": "§cYou have no permission!",
    "only_players": "§cOnly players can use this command!",
    "service_starting": "§7Service §b%service% §7is starting! §8[§6⌛§8]",
    "service_online": "§7Service §b%service% §7is now online! §8[§a✔§8]",
    "service_shutdown": "§7Service §b%service% §7has been shut down. §8[§c✘§8]"
  }
}
```

## Permissions

The Notify Addon uses the following permissions:

- **`polocloud.addons.notify.receive.starting`** - Receive service starting notifications
- **`polocloud.addons.notify.receive.online`** - Receive service online notifications
- **`polocloud.addons.notify.receive.shutdown`** - Receive service shutdown notifications

## Message Placeholders

The Notify Addon supports the following placeholders in messages:

- **`%service%`** - Service name
- **`%group%`** - Group name
- **`%player%`** - Player name
- **`%server%`** - Server name

## Getting Started

1. **Install the Addon** by placing the JAR file in `/polocloud/addons/`
2. **Restart PoloCloud** to load the addon
3. **Configure Messages** in the generated `notify-config.json` file
4. **Set up Permissions** for your players to receive notifications
5. **Test Notifications** by starting/stopping services

<Callout type="info">
**Pro Tip**: Use different permission levels to control who receives which types of notifications. For example, give staff members all notification permissions while giving regular players only online notifications.
